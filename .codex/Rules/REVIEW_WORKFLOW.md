# Review Workflow

1. Разбить файл на блоки:
- 1-6: C# / .NET
- 7-11: деревья
- 12-17: длинная арифметика, алгебра, многочлены, Карацуба
- 18-19: ДПФ / БПФ

2. Для каждого блока проверить:
- совпадает ли определение с источником;
- нет ли лишней уверенности;
- не смешаны ли разные модели и разные уровни абстракции;
- нет ли внутренних противоречий.

3. Маркировать выводы так:
- `ошибка`
- `спорная формулировка`
- `неполнота`
- `нормально`

4. Если отдельной статьи ИТМО нет, не выдумывать ее наличие.
Нужно прямо писать: `отдельную подтвержденную страницу не нашел`.
