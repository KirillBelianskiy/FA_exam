### 3. Практическая задача на C#

Реализовать битовый сдвиг модуля влево в `BetterBigInteger`.

Дано:
В `BetterBigInteger` модуль числа хранится как `uint[]` little-endian в основании `2^32`, знак хранится отдельно. Оператор `<<` должен использовать вспомогательный метод для сдвига модуля. Уже есть `TrimMagnitude` и `IsZeroMagnitude`.

Нужно:
Реализовать метод с предполагаемой сигнатурой `private static uint[] ShiftLeftMagnitude(ReadOnlySpan<uint> value, int shift)`, который возвращает модуль, сдвинутый влево на `shift` бит.

Ограничения:
`shift` неотрицателен. Нельзя использовать `System.Numerics.BigInteger`, строки и LINQ. Нужно отдельно учитывать сдвиг на целые 32-битные слова и остаточный битовый сдвиг.

Что должно работать:
- Сдвиг на `0` возвращает тот же модуль по значению.
- Сдвиг нуля возвращает `[0]`.
- При `shift >= 32` в начало little-endian массива добавляются нулевые младшие слова.
- Перенос старших битов должен попадать в следующее слово.

Мини-пример:
`value = [1]`, `shift = 33` должны дать `[0, 2]`, потому что `1 << 33 = 2 * 2^32`.
