### 3. Практическая задача на C#

Реализовать поразрядную операцию через дополнительный код.

Дано:
В `BetterBigInteger` знак хранится отдельно от модуля `uint[]` little-endian. Для поразрядных операций с отрицательными числами в проекте используется временное представление в дополнительном коде. Уже есть методы `GetSignedWordLength()`, `ToTwosComplement(int length)` и `FromTwosComplement(uint[] words)`.

Нужно:
Реализовать метод с предполагаемой сигнатурой `private static BetterBigInteger ApplyBitwise(BetterBigInteger left, BetterBigInteger right, Func<uint, uint, uint> operation)`. Метод должен применить операцию к словам двух чисел в дополнительном коде и вернуть обычный `BetterBigInteger`.

Ограничения:
Нельзя использовать `System.Numerics.BigInteger`, строки и LINQ. Оба числа нужно расширить до одинаковой длины, достаточной для знака результата. Метод должен бросать `ArgumentNullException`, если один из операндов равен `null`.

Что должно работать:
- Операции `&`, `|`, `^` должны переиспользовать один общий метод.
- Положительные и отрицательные числа должны обрабатываться одинаковым циклом по словам.
- Длина временных массивов должна учитывать знаковое расширение.
- Результат должен быть преобразован обратно из дополнительного кода.

Мини-пример:
Для `left = -1` и `right = 5` операция `&` должна вернуть `5`, потому что `-1` в дополнительном коде имеет все биты, равные `1`.
