### 3. Практическая задача на C#

Реализовать сложение модулей в `BetterBigInteger`.

Дано:
В классе `BetterBigInteger` длинное целое хранится как знак отдельно и модуль в массиве `uint[]` little-endian: `digits[0]` содержит младшие 32 бита, основание равно `2^32`. В проекте уже есть вспомогательный метод `TrimMagnitude(uint[] digits)`, который убирает ведущие нулевые слова и оставляет `[0]` для нуля.

Нужно:
Реализовать метод с предполагаемой сигнатурой `private static uint[] AddMagnitude(ReadOnlySpan<uint> left, ReadOnlySpan<uint> right)`. Метод должен сложить два неотрицательных модуля и вернуть нормализованный массив слов.

Ограничения:
Нельзя использовать `System.Numerics.BigInteger`, LINQ и перевод числа в строку. Нужно учитывать перенос между 32-битными словами через тип шире `uint`.

Что должно работать:
- Сложение массивов разной длины.
- Перенос из младшего слова в старшее.
- Добавление нового старшего слова, если после последней позиции остался перенос.
- Нормализация результата без лишних старших нулей.

Мини-пример:
`left = [uint.MaxValue]`, `right = [1]` должны дать `[0, 1]`, то есть `2^32`.
